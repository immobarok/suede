# Plan: Refactor About Page to Use API for Data Fetching

## Objective

Refactor the frontend about page to fetch company information (mission, values, story) via a new API endpoint instead of relying on server actions revalidation. Keep admin dashboard using server actions for content management. Ensure professional implementation with shadcn UI components, error handling, and loading states.

## Current State

- About page: `app/(main)/about/page.tsx` displays static company content managed via server actions (`app/actions/about-content.ts`).
- Admin dashboard: Uses server actions for CRUD operations on about content, stored in `db/schema/about.ts` (aboutContentUploads table).
- Data flow: Server actions update DB and revalidate paths for dynamic content.
- Shadcn UI: Already integrated throughout the app.

## Requirements

- Create a new API route to fetch about content.
- Update about page to fetch data server-side from the API.
- Maintain existing admin functionality (server actions).
- Use shadcn components for any new UI elements (e.g., loading/error states).
- Follow professional practices: TypeScript, error handling, loading indicators.

## Steps

1. **Analyze current about page implementation**
   - Review `app/(main)/about/page.tsx` and components to understand data usage.
   - Confirm data structure from `db/schema/about.ts`.

2. **Create API endpoint**
   - Add `app/api/about/route.ts` (GET) to fetch about content from database.
   - Use Drizzle queries to retrieve data.
   - Return JSON response with proper typing.

3. **Update about page**
   - Modify `app/(main)/about/page.tsx` to fetch from `/api/about` server-side.
   - Handle loading and error states with shadcn components (e.g., Skeleton, Alert).
   - Ensure SSR for SEO.

4. **Add professional touches**
   - Implement TypeScript interfaces for API responses.
   - Add error boundaries or try-catch for API calls.
   - Test the integration to ensure data displays correctly.

5. **Verify admin dashboard unchanged**
   - Confirm server actions in admin still work for content updates.
   - Revalidate paths still trigger on updates.

## Risks

- Potential downtime if API fails; implement fallback to cached/static data.
- Ensure API performance doesn't impact page load times.
